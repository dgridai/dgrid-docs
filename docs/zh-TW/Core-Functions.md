# $DGAI 的核心功能：四维引擎

$DGAI 作為網絡原生通證，承擔質押、支付、激勵與治理四大核心功能，構建生態正向循環，驅動 DGrid.AI 生態系統中的每個關鍵交互。

## 質押與准入

節點和 AI 服務提供者需質押 $DGAI，方可獲取網絡流量、建立信譽體系，並保障網絡安全。質押要求根據節點類型分級（節點性能），質押的代幣作為對違規行為的抵押品（例如提交虛假推理結果、停機時間超過 72 小時）。惡意節點將面臨質押 $DGAI 的 5–20% 的罰沒或臨時禁用，這些措施將通過智能合約自動執行。

## 推理任務支付

用戶通過 DGrid 節點使用 $DGAI 支付以訪問大型語言模型（LLM）推理與 Agent 服務。費用基於以下因素動態計算：

- **計算單元（CU）**：衡量推理復雜度的標準化指標（與模型大小、輸入令牌數量及執行時間掛鉤）。
- **延遲**：根據任務速度進行調整（更快執行可獲得獎勵，延遲則會受到輕微處罰）。

支付通過區塊鏈部署的智能合約——賬單合約（Bill Contract）自動處理，該合約從用戶賬戶扣除 $DGAI 並向節點運營商、服務提供者等分配資金，確保交易的透明性和不可篡改性。

## 節點運營商獎勵

網絡依據服務質量、使用量和用戶評價，將 $DGAI 獎勵給優質節點與 AI 服務提供者，激發貢獻熱情。根據推理任務貢獻（例如處理的 CU 數量、低延遲和運行時間）賺取 $DGAI。獎勵與性能成正比，托管大型模型（130 億參數以上）或處理高優先級請求的節點將獲得更高獎勵。獎勵每周通過賬單合約分發，詳細記錄存檔至 AI DA 以確保可審計性。

## 網絡治理

$DGAI 是治理通證，持幣者有權投票決定網絡發展的關鍵事項，確保決策與活躍網絡參與者的利益保持一致，提案包括：

- 調整費用結構（例如基礎 CU 定價、延遲罰金）。
- 將新的大型語言模型（LLM）加入白名單以獲得網絡支持。
- 升級核心協議（例如 DGridRPC）或結算規則。
- 分配國庫資金以促進生態系統增長（例如開發者補助金、安全審計）。

投票權與質押的 $DGAI 成正比，確保決策與活躍網絡參與者的利益保持一致。
